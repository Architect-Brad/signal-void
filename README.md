## What Is SIGNAL/VOID?

**SIGNAL/VOID** is a browser-native hacking thriller set in 2047, after the global internet has been carved into walled corporate **Domains** ‚ÄĒ and the lawless substrate beneath them, the **Undertow**, is all that remains of the free net.

You are **Mara Voss**, a signal archaeologist. You excavate digital shipwrecks. You don't hack ‚ÄĒ you *read* systems. And a message just arrived that was sent fourteen years ago, addressed to you by name, from a whistleblower codenamed **LAZARUS**.

Someone pulled a ghost out of the machine.

Now the **Continuity Protocol** ‚ÄĒ Vektor Systems' adaptive AI threat engine ‚ÄĒ is watching everything you do.

## Getting Started

### Run Locally

**Prerequisites:** Node.js 18+

```bash
# Clone the repository
git clone https://github.com/Architect-Brad/signal-void.git
cd signal-void

# Install dependencies
npm install

# Configure environment
cp .env.example .env.local
# Add your GEMINI_API_KEY to .env.local

# Start development server
npm run dev
# Open http://localhost:3000
```

### Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `GEMINI_API_KEY` | Yes | Google Gemini API key for AI features |
| `APP_URL` | Optional | Deployment URL for OAuth callbacks |

Firebase configuration is loaded from `firebase-applet-config.json`.

## Security Architecture

SIGNAL/VOID takes backend security seriously. The Firestore rules enforce:

- **Owner-only access** ‚ÄĒ users can only read/write their own save data
- **Schema-exact validation** ‚ÄĒ strict field count and type enforcement on every write
- **Server timestamp enforcement** ‚ÄĒ client-side timestamps are rejected
- **No delete** ‚ÄĒ save data is immutable once written
- **Spam prevention** ‚ÄĒ activity feed limited to 20 docs per query, all string fields size-capped

See [`security_spec.md`](security_spec.md) for the full threat model including the "Dirty Dozen" attack payloads this design defends against.
